# Security

## Reporting Vulnerabilities

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. Email the maintainers or open a private security advisory on GitHub.
3. Include a clear description of the vulnerability and steps to reproduce.
4. Allow time for a fix before public disclosure.

## Best Practices

- Never commit `.env` files, API keys, or access tokens.
- Use environment variables for org-specific configuration.
- Replace all `YOUR_ORG_ID` and `YOUR_CONTACT_ID` placeholders before deploying to production.
